# Tract Map App

This app builds a standalone tract-level interactive map from the local data
pipeline outputs under `/Users/jinyang/Desktop/processing/data_pipeline/tract`.

## What It Produces

- `data/tracts_joined.geojson`: tract geometry joined to selected indicator fields
- `data/indicators.json`: indicator catalog and precomputed styling metadata
- `data/census_taxonomy.json`: category browser structure derived from `dictionary_副本.xlsx`
- `data/cbsa_stats.json`: CBSA comparison stats for the selected indicators
- `data/tiles/tracts/*`: static vector tile directory generated by `tippecanoe`
- `web/*`: standalone map page

## Project Layout

- `scripts/indicator_definitions.py`: shared indicator catalog and cleaning rules
- `scripts/build_indicator_metadata.py`: builds `indicators.json`
- `scripts/build_census_taxonomy.py`: builds `census_taxonomy.json`
- `scripts/build_cbsa_stats.py`: builds `cbsa_stats.json`
- `scripts/build_joined_geojson.py`: builds `tracts_joined.geojson`
- `scripts/build_tiles.sh`: generates static vector tiles from the joined GeoJSON
- `scripts/build_all.sh`: runs the full data build pipeline

## Prerequisites

- `python3`
- `pandas`
- `tippecanoe`

## Build

From `/Users/jinyang/Desktop/processing/tract_map_app`:

```bash
./scripts/build_all.sh
```

That will:

1. Generate indicator metadata
2. Generate category taxonomy
3. Generate CBSA comparison stats
4. Join tract geometry and selected attributes
5. Build static vector tiles into `data/tiles/tracts`

## Local Preview

From `/Users/jinyang/Desktop/processing/tract_map_app`:

```bash
python3 scripts/serve_local.py --port 8010
```

Then open:

`http://localhost:8010/web/`

## Notes

- The first full build may take a while because it processes nationwide tract
  geometry.
- `hcv_*` fields use `-4` in the master table as a no-data sentinel; the build
  scripts convert those values to `null`.
- The front end is intentionally static and framework-free so it can be hosted
  as plain files.
- The generated `.pbf` vector tiles are gzip-compressed, so use
  `scripts/serve_local.py` for local preview rather than `python -m http.server`.
